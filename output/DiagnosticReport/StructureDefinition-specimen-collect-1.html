SD.10.1 Formal Views of Profile Content

The official URL for this profile is: 

http://ns.electronichealth.net.au/ci/fhir/4.0/StructureDefinition/specimen-collect-1

The purpose of this profile is to represent key elements of clinical relevance of a collected specimen to accompany a diagnostic observation issued by the diagnostic service provider for the electronic exchange of diagnostic test observations between healthcare providers, and between healthcare providers and the My Health Record system infrastructure in Australia.

Must Support

In the context of this profile Must Support SHALL be interpreted as follows:

  • The system SHALL be able to include the element when authoring a resource where data is available

  • The system SHALL be able to store and retrieve the element

  • The system SHALL be able to process resources containing the element

This profile builds on AUBaseSpecimen.

This profile was published on Thu May 14 13:56:08 AEST 2020 as a draft by Australian Digital Health Agency.

SD.10.2 - Known Issues

Reference Description Issue No.
self collection (not currently support) It is not understood how self collection can be supported in this profile. There is no element or core extension available for Specimen that identifies a patient or related person as a collector.

Self collection of bloods or other material including sent by mail kits cancer screening are to be a supported usage scenario. 		See ci-fhir-r4/issues/24, and jira.hl7.org/browse/FHIR-26922
Support for screening programs initiated requests Not yet clear how requests that are initiated by a national program, e.g. National Bowel Cancer Screening Program, are supported with this model - work in progress. See ci-fhir-r4/issues/29
type (coding terminology enforcement) It is intended that the slice from HL7 AU Base Specimen coding:snomedSpecimenType should be enforced if any coding is provided. In other words, either text or a code from this value set is required. This is a work in progress. See ci-fhir-r4/issues/24
subject (Reference type too open) The Reference type is too open, we need to ensure that either a reference conforming to the profiles or a logical reference via identifier is supplied - work in progress. See ci-fhir-r4/issues/24
collection.collector (Reference type too open) The Reference type is too open, we need to ensure that either a reference conforming to the profiles or a logical reference via identifier is supplied - work in progress. See ci-fhir-r4/issues/24
collection.method (coding terminology enforcement) It is intended that the slice from HL7 AU Base Specimen coding:snomedSpecimenCollectionProcedure should be enforced if any coding is provided. In other words, either text or a code from this value set is required. This is a work in progress. See ci-fhir-r4/issues/24
collection.bodySite (coding terminology enforcement) It is intended that the slice from HL7 AU Base Specimen coding:snomedBodySite should be enforced if any coding is provided. In other words, either text or a code from this value set is required. This is a work in progress. See ci-fhir-r4/issues/24
